SQL (Structured Query Language, strukturalny język zapytań).
Użycie SQL, zgodnie z jego nazwą, polega na zadawaniu zapytań 
do bazy danych. 
Zapytania można zaliczyć do jednego z czterech głównych podzbiorów:
* SQL DML(ang. Data Manipulation Language-język manipulacji danymi),
* SQL DDL (ang. Data Definition Language – język definicji danych),
* SQL DCL (ang. Data Control Language – język kontroli nad danymi).
* SQL DQL (ang. Data Query Language – język definiowania zapytań

DML (Data Manipulation Language) służy do wykonywania operacji 
na danych – do ich umieszczania w bazie, kasowania, przeglądania 
oraz dokonywania zmian. 
Najważniejsze polecenia z tego zbioru to:
* INSERT – umieszczenie danych w bazie,
* UPDATE – zmiana danych,
* DELETE – usunięcie danych z bazy.
Dane tekstowe muszą być zawsze ujęte w pojedynczy cudzysłów (')

Dzięki DDL (Data Definition Language) można operować na strukturach, 
w których dane są przechowywane – czyli np. dodawać, zmieniać 
i kasować tabele, bazy, użytkowników, widoki itp. 
Najważniejsze polecenia tej grupy to:
* CREATE (np. CREATE TABLE, CREATE DATABASE, CREATE USER, ...) 
		– utworzenie struktury (bazy, tabeli, indeksu itp.),
* DROP (np. DROP TABLE, DROP DATABASE, DROP VIEW, ...) 
		– usunięcie struktury,
* ALTER (np. ALTER TABLE ADD COLUMN ...) 
		– zmiana struktury (dodanie kolumny do tabeli, 
			zmiana typu danych w kolumnie tabeli).
* TRUNCATE - powoduje usunięcie danych z tabeli 
	(np. TRUNCATE TABLE klienci)

DCL (Data Control Language) ma zastosowanie do nadawania uprawnień 
do obiektów bazodanowych. Najważniejsze polecenia w tej grupie to:
* GRANT – pozwala użytkownikowi lub roli na wykonywanie operacji 
	określonej przez nadane uprawnienie 
	(np. GRANT ALL PRIVILEGES ON FIRMA TO PIOTR WITH GRANT OPTION 
	– przyznanie wszystkich praw do tabeli FIRMA użytkownikowi 
	PIOTR z opcją pozwalającą mu nadawać prawa do tej tabeli).
* REVOKE – usuwa uprzednio nadane lub odmówione uprawnienie 
	(np. REVOKE ALL PRIVILEGES ON FIRMA FROM PIOTR 
	– odebranie użytkownikowi wszystkich praw do tabeli FIRMA).
* DENY - odmawia użytkownikowi lub roli określonego uprawnienia 
	i zapobiega odziedziczeniu go po innych rolach

DQL (Data Query Language) - to instrukcje, za pomocą których możesz 
	otrzymać z bazy określone dane. 
	Najważniejszym poleceniem jest tutaj SELECT.

TCL (ang. Transactional Control Language) 
		– „język kontroli transakcji”
* BEGIN - - otwiera transakcję
* COMMIT -zatwierdza transakcję
* ROLLBACK- cofnij transakcję
* SAVEPOINT – zapis punkt transakcj

Dodajemy podczas tworzenia tabeli (CREATE) lub później (ALTER)

---W poleceniu CREATE---

1. W linii tworzącej kolumnę:

PRIMARY KEY - Klucz główny
UNIQUE - Unikatowe wartości
REFERENCES <tabela(kolumna/y)>[ON DELETE <opcja>[ON UPDATE <opcja> - klucz obcy
(-opcja: CASCADE, RESTRICT, SET NULL, NO ACTION, SET DEFAULT)
DEFAULT <wartość> - wartość wstawiana domyślnie
NOT NULL - nie może być pusta
AUTO_INCREMENT [=<wartość początkowa>] - autonumerowanie

2. poniżej definicji kolumn:
PRIMARY KEY (<nazwa kolumny>)
UNIQUE (<nazwa kolumny>)
CHECK <(warunek)> - tylko poniżej
FOREIGN KEY (<nazwa/y kolumny>) REFERENCES <tabela(kolumna/y)>
[ON DELETE <opcja>[ON UPDATE <opcja>]
(-opcja: CASCADE, RESTRICT, SET NULL, NO ACTION, SET DEFAULT)
*<nazwa/y kolumny> - to nazwa kolumny w tabeli podrzędnej która ma być kluczem obcym;
*<tabela(kolumna/y)> - to nazwa tabeli nadrzędnej i nazwa kolumn/y j.w. tej tabeli z która jest powiązana kolumna/y klucza obcego;

CONSTRAINT <nazwa> <OGRANICZENIE j.w> - dodanie ograniczenia z nazwą


CASCADE: Usuń lub zaktualizuj wiersz z tabeli nadrzędnej i automatycznie usuń lub zaktualizuj pasujące wiersze w tabeli podrzędnej. Obsługiwane są zarówno ON DELETE CASCADE, jak i ON UPDATE CASCADE. Pomiędzy dwiema tabelami nie należy definiować kilku klauzul ON UPDATE CASCADE, które działają na tę samą kolumnę w tabeli nadrzędnej lub w tabeli podrzędnej.

SET NULL: Usuń lub zaktualizuj wiersz z tabeli nadrzędnej i ustaw kolumnę lub kolumny klucza obcego w tabeli podrzędnej na NULL. Obsługiwane są zarówno klauzule ON DELETE SET NULL, jak i ON UPDATE SET NULL. Jeśli określisz akcję SET NULL, upewnij się, że kolumny w tabeli podrzędnej nie zostały zadeklarowane jako NOT NULL.

RESTRICT: Odrzuca operację usuwania lub aktualizacji tabeli nadrzędnej. Określenie RESTRICT (lub NO ACTION) JEST RÓWNOZNACZNE Z POMINIĘCIEM KLAUZULI on DELETE LUB on UPDATE.

NO ACTION: Słowo kluczowe ze standardowego SQL.
W MySQL odpowiednik RESTRICT. Serwer MySQL odrzuca operację usuwania lub aktualizacji tabeli nadrzędnej, jeśli w tabeli do której się odwołuje, znajduje się powiązana wartość klucza obcego. Niektóre systemy baz danych mają odroczone sprawdzanie, a NO ACTION to odroczone sprawdzanie. W MySQL ograniczenia klucza obcego są sprawdzane natychmiast, więc NO ACTION jest tym samym co RESTRICT.

SET DEFAULT: Ta akcja jest rozpoznawana przez parser MySQL, ale InnoDB odrzuca definicje tabel zawierające klauzule ON DELETE SET DEFAULT lub ON UPDATE SET DEFAULT.

W przypadku nieokreślonego polecenia ON DELETE lub ON UPDATE domyślną akcją jest zawsze RESTRICT.

CREATE TABLE produkty1
(
numer int PRIMARY KEY,
nazwa varchar(10) UNIQUE,
cena numeric(9,2) NOT NULL DEFAULT 10,
opis varchar(100) NOT NULL,
nr_kategorii int REFERENCES kategorie(numer)
);

definicji kolumn:
CREATE TABLE produkty2
(
numer INT,
nazwa varchar(10),
cena decimal(9,2) NOT NULL DEFAULT 10,
opis varchar(100) NOT NULL,
nr_kategorii INT,
PRIMARY KEY (numer),
UNIQUE (nazwa),
CHECK (cena < 1000),
FOREIGN KEY (nr_kategorii) REFERENCES kategorie(numer)
);

Tworzymy tabele kategorie
CREATE TABLE kategorie
( numer INT PRIMARY KEY AUTO_INCREMENT, nazwa varchar(10) NOT NULL );
